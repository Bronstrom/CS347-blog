Web frameworks are being used all across the web and around us everyday. In a way, they are the way of the future when it comes to developing web applications!

In What Way are We Framing the Web?

In a basic sense, frameworks are code libraries, that tend to be open source. They can help achieve structure and give additional features that may take a ton of work to put together from scratch. They can nicely provide a place to start with an ideal set of configurations for a project. When we speak in terms of web frameworks - they can be in relation to frontend, backend, or even interface frameworks.

What's the Point of Using a Framework?

Let's put it this way: you don't have to build an application on a web framework, although, it will likely be a lot more tedious and there will be a lot of duplication in terms of features. Without the use of a framework to build an application, you will be writing duplicate logic a lot of the time. Frameworks give a lot of tools to developers, which can enhance and speed up the efficiency of an application. With frameworks, there is often a trade off between abstraction of framework code and a limitation of some flexibility versus the amount of complexity of code. Although, some frameworks may have a bit of a learning curve, in the long run they will can be huge time savers.

Can You Elaborate on the Types of Frameworks?

As mentioned above, there are three main types of web frameworks: frontend, backend, and interface frameworks. In terms of web applications, the frontend consists of the interactive parts of the page, as the backend consists of the server and database as well as the code interacting with both, while the interface is the stylized and parts polishing the page. I'll mention a few popular frameworks for each category of a web application and there benefits (& cons):

Frontend Frameworks

ReactJS

  • Bundles code into components
  • Reusability of code due to organization patterns
  • Steep learning curve

Angular

  • Built be Google & is well supported
  • Application scalability & reusability
  • Steep learning curve

Vue

  • More styling Options
  • Fast runtime (optimizes parts of Angular & React) & flexible
  • Less widely adopted

Backend Frameworks

Ruby on Rails

  • Ruby-based
  • More functionality out of box
  • Very opinionated
  • Ruby may take more time to run than other languages

Django

  • Python-based
  • More functionality out of box
  • Between very opinionated & less structured
  • Front-end incorporation can be difficult

Flask

  • Python-based
  • More customizable
  • Less opinionated
  • Less functionality out of box

Interface Frameworks

Bootstrap

  • Well-known & professional
  • Easy learning curve

Materialize

  • More styling Options
  • Closely follows Google's Material stylizing guide

Semantic

  • Lots of built-in parts
  • Heavily customizable

When Choosing Frameworks What Should come into Consideration?

Choosing frameworks for a project, is a vital step to the development life cycle. Often times when a working with a development team (especially when under tight time constraints), learning curve of a framework, integration of libraries, familiarity of languages, and desired features should come into mind. Often time, the popularity of frameworks should be considered to either meet or go beyond client and industry standards.

With So Many Frameworks for Parts of An Application, Are There Ideal/Common Pairings?

While your goals should be assessed first when choosing frameworks, standards are also a huge consideration. In terms of more ideal pairings between front-end, back-end, & interface frameworks, there are some more compatible and practical pairings. I'll mention a couple below with their benifits:

  • ReactJS and Express/Node.js - Both based on JavaScript languages and ease of running React code on NodeJS environment
  • Angular and Express/Node.js - Both support Typescript and Node package manager provides "ng CLI" (the Angular command-line interface)
  • Angular and Spring Boot - Spring boot supports Angular's two-way (bidirectional) binding and both combined provide a secure user experience
  • React Native and Django - Runs fast, secure & requires less code compared to other frameworks
  • Vue and Lararvel - Coupled well with each other and Laravel implements Vue in an easy way a scaffolding (basic skeleton design) UI
