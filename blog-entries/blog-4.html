Week 4 (9/24/21): CSS Animation

Straying away from broader concepts of web dev, this week we will touch on a more visual focused aspect: animation using CSS.

What is Animation to Begin With?

Simply put, the term animate means to "bring to life". Animation involves a number of pictures, known as frames, and when put together can create a sense of movement. These moving pictures can be used to entertain an audience by telling a story, inspire similar creations, and potentially inform an audience better by it's ability to guide them with motion. In web design, it can be used enhance the experience of an interface, as well as give some visual flare and life to a relatively static page.

Where does CSS Come Into Play with Animation?

CSS supports the following features allowing for animation to be used on web pages:

  1. Keyframes: define the significant turning points, or stages, in the style of animation
  2. Properties of animation: assign keyframes a specified CSS element and the way it is animated

Where do I Get Started with CSS Animation?

One of the simpilist and common forms of animation on webpages are transitions. Transitions can be used to create a visual effect when interacting between two types of states. They allow visual effects like fades, slides, and scaling effects between states of a broader property. With the transition property, give a specified property to animate (by default "all" properties), the amount of time for the transition (typically in seconds), and to "ease" the transition (no ease by default). See the first two examples below for transition based animations.

With the animation property, you can create more complex forms of motion with keyframes, called motion paths. Specify the given animation property with the animation name and the amount of time. You can also use the animation-iteration-count to loop the animation a certain number of times. To create an animation, specify it with "@keyframe" followed by it's name. Inside, are our listed keyframes, specifically our immbed modifications to properties at specific periods of time. To state a keyframe, the first keyframe can be stated as "from" (or 0%) and the last as "to" (or 100%). For in-between frames, specify percentages at when the keyframe should take place. See the third example below for this this type of animation.

Examples

HTML

            

                <!DOCTYPE html>
                <html>

                <head>
                    <title>Blog #4 Example</title>

                    <meta charset="utf-8">
                    <meta name="description" content="Blog 4 Example">
                </head>

                <body>
                    <div id="purple-circle">Press me!</div>
                    <div id="green-square">Hover over me!</div>
                    <div class="polygon-container"><div id="orange-polygon"></div><p>Watch me!</p></div>

CSS

            

                body {
                    margin: 250px;
                    display: flex;
                    justify-content: space-between;
                }
                
                /* Set same properties between elements */
                #purple-circle, #green-square, #orange-polygon {
                    height: 150px;
                    width: 150px;
                    position: relative;
                    text-align: center;
                    white-space: nowrap;
                    line-height: 150px;
                    color: white;
                    user-select: none;
                }
                
                #purple-circle {
                    background-color: blueviolet;
                    border-radius: 50%;
                    transition: 0.5s;
                }
                
                /* When pressed (active) the font & background color will change */
                #purple-circle:active {
                    background-color: yellow;
                    color: royalblue;
                }
                
                
                #green-square {
                    background-color: limegreen;
                    transition: 1.5s ease;
                }
                
                /* When hovered over the shape will scale & rotate */
                #green-square:hover {
                    transform: rotateZ(135deg) scale(2);
                }
                
                .polygon-container {
                    display: flex;
                    flex-direction: column;
                    align-items: center;
                    
                }
                .polygon-container p {
                    padding-top: 10px;
                }

                #orange-polygon {
                    width: 0px;
                    height: 0px;
                    line-height: 25px;
                    border-left: 100px solid transparent;
                    border-right: 100px solid transparent;
                    border-bottom: 150px solid coral;
                    animation: unpredictable-shift 3s;
                    animation-iteration-count: infinite;
                }
                
                /* Animation for orange-polygon - Is it really that unpredicatable?! */
                @keyframes unpredictable-shift {
                    0% {
                        transform: translate(0px, 0px);
                    }
                    22% {
                        transform: translate(25px, 15px);
                        width: 0px;
                        /* Transform into triangle */
                        line-height: 25px;
                        border-left: 100px solid transparent;
                        border-right: 100px solid transparent;
                    }
                    41% {
                        transform: translate(-20px, -5px);
                    }
                    64% {
                        transform: translate(10px, 15px);
                        /* Transform into trapezoid */
                        width: 100px;
                        border-left: 50px solid transparent;
                        border-right: 50px solid transparent;
                    }
                    75% {
                        transform: translate(-5px, -20px);
                    }
                    90% {
                        transform: translate(-15px, 10px);
                    }
                    100% {
                        transform: translate(0px, 0px);
                    }
                }

Are There any Major Concerns with Animation on a Webpage?

We should always be considerate of the performance of a page when containing animation. The load times on a page can take a while if containing tons of animation, especially when stacked on top of eachother.

Also, you should be wary that not all CSS properties can be animated. To see what is supported, check out MDN's list of animatable properties.
